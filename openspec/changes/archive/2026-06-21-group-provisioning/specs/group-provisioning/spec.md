# group-provisioning Delta Specification

## ADDED Requirements

### Requirement: Создание недостающих групп

`census apply` ДОЛЖЕН (MUST) создавать группы, на которые ссылается состав ролей
(`payload.groups`) или декларативный блок `[[group]]`, если их нет в системе, и заносить
созданные группы в реестр managed. Создание групп ДОЛЖНО (MUST) выполняться до создания и
обновления роль-учёток (членство требует существующей группы). Группа, которая уже существует,
но отсутствует в реестре managed (пред-существующая/чужая), НЕ ДОЛЖНА (MUST NOT) изменяться или
заноситься в реестр — Census её не создавал.

#### Scenario: Группа роли отсутствует
- **WHEN** apply применяет роль с группой, которой нет в системе
- **THEN** группа создаётся до учётки, заносится в реестр, учётка получает членство

#### Scenario: Пред-существующая чужая группа
- **WHEN** требуемая роль ссылается на группу, уже существующую и не созданную Census
- **THEN** Census не трогает группу и не заносит её в реестр, но назначает членство

### Requirement: Стабильный GID по пину

Если декларация пинит GID группы (`[[group]] gid`), `census apply` ДОЛЖЕН (MUST) создавать
группу с этим GID. При конфликте (GID уже занят другой группой) apply ДОЛЖЕН (MUST) отказывать
с диагностикой, а не перезаписывать. Census НЕ ДОЛЖЕН (MUST NOT) перенумеровывать GID
существующей группы на лету.

#### Scenario: Пин GID
- **WHEN** декларация задаёт группе явный gid, которого нет в системе
- **THEN** группа создаётся с этим gid

#### Scenario: Конфликт GID
- **WHEN** пинённый gid уже принадлежит другой группе
- **THEN** apply отказывает с диагностикой, ничего не перезаписывая

### Requirement: Удаление осиротевших managed-групп

`census apply` ДОЛЖЕН (MUST) удалять группу, ТОЛЬКО если она присутствует в реестре managed и
больше не требуется ни одной ролью и не объявлена в декларации. Удаление групп ДОЛЖНО (MUST)
выполняться после удаления роль-учёток (нельзя снять группу, пока у неё есть члены). Группу вне
реестра managed Census удалять НЕ ДОЛЖЕН (MUST NOT).

#### Scenario: Осиротевшая managed-группа
- **WHEN** managed-группа (созданная Census) больше не упоминается ни ролью, ни декларацией
- **THEN** apply удаляет её после удаления учёток

### Requirement: Откат групповых изменений

Изменения групп ДОЛЖНЫ (MUST) входить в тот же full-file backup (`/etc/group`, `/etc/gshadow`),
что и учётки; при сбое любой фазы apply ДОЛЖЕН (MUST) восстановить группы из снапшота вместе с
остальными auth-базами (атомарность §6).

#### Scenario: Сбой после создания группы
- **WHEN** фаза после создания группы падает
- **THEN** созданная группа откатывается из снапшота, состояние согласовано
