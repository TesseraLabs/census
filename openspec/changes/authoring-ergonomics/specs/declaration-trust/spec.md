# declaration-trust Delta Specification

## ADDED Requirements

### Requirement: Версия формата схемы декларации (schema)

Декларация ДОЛЖНА (MUST) нести обязательное поле `schema` (целое ≥ 1) — версию формата
парсера, отличную от `version` (anti-rollback контента). Парсер ДОЛЖЕН (MUST) проверять
`schema` ПЕРВЫМ, до любой прочей валидации: при `schema` больше поддерживаемой крейтом
(`SUPPORTED_SCHEMA`) парс ДОЛЖЕН (MUST) отказывать с понятной ошибкой (`schema N не
поддержан, максимум M`), а не падать позже на `deny_unknown_fields`. Отсутствие `schema`
ДОЛЖНО (MUST) быть ошибкой парса. `schema` меньше поддерживаемой зарезервирована под будущую
ветку миграции формата.

#### Scenario: Поддерживаемая schema
- **WHEN** декларация содержит `schema = 1`, а `SUPPORTED_SCHEMA = 1`
- **THEN** парс проходит проверку версии и продолжает обычную валидацию

#### Scenario: Неподдерживаемая будущая schema
- **WHEN** декларация содержит `schema`, большую `SUPPORTED_SCHEMA`
- **THEN** парс отказывает с ошибкой о неподдерживаемой версии формата, ДО прочей валидации

#### Scenario: Отсутствует schema
- **WHEN** декларация не содержит поля `schema`
- **THEN** парс отказывает (поле обязательно)

#### Scenario: schema проверяется раньше прочих ошибок
- **WHEN** декларация имеет и неподдерживаемую `schema`, и иную ошибку валидации (например, инвертированный `uid_range`)
- **THEN** возвращается ошибка неподдерживаемой `schema` (приоритет проверки формата)
